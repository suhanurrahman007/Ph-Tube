Q1. Discuss the scope of var, let, and const

Ans : var: Function-scoped. Accessible within the function they're declared in or globally if declared outside a function.
let: Block-scoped. Accessible only within the block (e.g., loop or conditional) where they're declared.
const: Block-scoped like let, but once assigned, its value cannot be changed.
As best practice, prefer const by default, use let when you need to reassign a variable, and avoid var in modern JavaScript to prevent unexpected behaviors due to its function scope.

Q2. Tell us the use cases of null and undefined

Ans : undefined: Indicates a variable has been declared but not yet assigned a value. Default value for function parameters that aren't provided. Object properties that don't exist return undefined. null: Represents an intentional absence of any value or object. Used to indicate "no value" or "empty" in contexts where it's meaningful (e.g., a database record that doesn't exist). In practice, undefined often arises unintentionally due to language mechanics, while null is usually set deliberately by developers.

Q3. What do you mean by REST API?

Ans : REST API (Representational State Transfer Application Programming Interface) is a set of conventions for building web services that allow systems to communicate over a network using stateless requests, typically over HTTP. It uses standard methods (like GET, POST, PUT, DELETE) to perform CRUD operations and often exchanges data in JSON or XML format.
